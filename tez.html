TEZ Language

TEZ is a C-like programming language that compiles directly to 32-bit x86 machine code inside the kernel. No libc, no runtime — pure bare-metal execution.

Source files use the .tez extension. Run with run prog.tez or use the interactive REPL with tez.

Compiled x86 32-bit output In-kernel JIT No runtime deps C-like syntax

Output with cons.out()

int n = 100;
cons.out(n);   // prints: 100

str s = "hi";
cons.out(s);   // prints: hi

cons.out() automatically adds a newline after output. For integers, prints decimal representation. For strings, prints the string content.

⚙ Functions
_
□
×

Declaration syntax

Format: return_type F name(type param, ...) { body }

// function with return value
int F add(int a, int b) {
  rtn a + b;
}

// void function
nr F greet(int n) {
  cons.out(n);
}

Calling functions

int result = add(10, 20);
cons.out(result);
// output: 30

greet(42);
// output: 42

Functions must be declared before they are called (top-down order). Max 8 parameters. Max 32 functions per program.

📄 Full Example Program
_
□
×

fibonacci.tez

// compute nth fibonacci number
int F fib(int n) {
  if (n <= 1) {
    rtn n;
  }
  rtn fib(n - 1) + fib(n - 2);
}

int i = 0;
while (i < 10) {
  cons.out(fib(i));
  i = i + 1;
}

counter.tez

// count down from 10
int n = 10;
while (n > 0) {
  cons.out(n);
  n = n - 1;
}

// for loop example
for (int i = 1;
     i <= 5;
     i = i + 1) {
  int sq = i * i;
  cons.out(sq);
}

How to run

# create the file with tuze editor
> tuze fibonacci.tez

# or create with mkf for short programs
> mkf test.tez int x = 5; cons.out(x);

# run it
> run fibonacci.tez
🔧 How TEZ Compiles
_
□
×
source.tez
   ↓ tez_lexer.c
Token stream [TK_INT, TK_IDENT("x"), TK_ASSIGN, TK_INT_LIT(42), ...]
   ↓ tez_parser.c
AST: ND_VAR_DECL(x) → ND_INT(42)
        ND_CONS_OUT → ND_IDENT(x)
   ↓ tez_codegen.c
x86: push ebp; mov ebp,esp; sub esp,4
       mov eax,42; mov [ebp-4],eax
       mov eax,[ebp-4]; push eax; call print_int
       mov esp,ebp; pop ebp; ret
   ↓ direct call
output: 42

The compiled code lives in a static 8KB buffer in kernel BSS. Variables are allocated on the stack (ebp frame). cons.out() calls kernel's vga_putchar/vga_print directly via absolute address. No memory protection — runs in ring 0, flat 32-bit address space.
