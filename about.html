About TEZ_OS

TEZ_OS is a 32-bit x86 operating system written entirely from scratch in C and x86 Assembly. It runs on real hardware or any x86 emulator (QEMU, VirtualBox, Bochs).

The project started as a learning exercise — understanding how computers actually work at the lowest level. No tutorials were fully followed. Every component was designed and debugged manually.

What it does

The bootloader lives in sector 0 of the disk — exactly 512 bytes, ending with the magic signature 0xAA55. BIOS loads it to address 0x7C00 and jumps to it.

It includes a full BPB (BIOS Parameter Block) so the disk is recognized as a valid FAT12 floppy.

After printing a loading message, it uses INT 13h / AH=42h (Extended Read with DAP) to load the kernel from LBA 50 into memory at 0x10000.

Then it sets up the GDT (Global Descriptor Table) with code and data segments, sets bit 0 of CR0 to enable Protected Mode, and does a far jump to 32-bit code.

Key code

; DAP — Disk Address Packet
dap:
  db 0x10     ; size
  db 0
  dw 128      ; sectors to read
  dw 0x0000   ; buffer offset
  dw 0x1000   ; buffer segment → 0x10000
  dq 50       ; start LBA

; switch to protected mode
cli
lgdt [gdt_descriptor]
mov eax, cr0
or eax, 1
mov cr0, eax
jmp 0x08:pm_entry

IDT (Interrupt Descriptor Table)

The IDT maps interrupt numbers to handler functions. Each entry is 8 bytes containing the handler address, code segment selector, and flags.

TEZ_OS sets up 256 IDT entries (all initially zeroed), then installs handlers for IRQ0 (PIT timer) at INT32 and IRQ1 (keyboard) at INT33.

PIC Remapping

The original x86 PIC maps hardware IRQs to INT 8–15, which conflicts with CPU exceptions. TEZ_OS remaps the PIC so IRQ0 = INT32, IRQ1 = INT33, etc.

Mask register: 0xFC (11111100) — allows only IRQ0 and IRQ1.

// PIC remap: ICW1–ICW4
outb(PIC1_COMMAND, 0x11);
outb(PIC2_COMMAND, 0x11);
outb(PIC1_DATA, 0x20);  // IRQ0=INT32
outb(PIC2_DATA, 0x28);  // IRQ8=INT40
outb(PIC1_DATA, 0x04);
outb(PIC2_DATA, 0x02);
outb(PIC1_DATA, 0x01);
outb(PIC2_DATA, 0x01);
// allow IRQ0 + IRQ1 only
outb(PIC1_DATA, 0xFC);
outb(PIC2_DATA, 0xFF);

FAT12 Layout

SectorContent
0Boot sector (BPB)
1–9FAT copy 1
10–18FAT copy 2
19–32Root directory (224 entries)
33+Data clusters
50+Kernel binary (seek=50)

12-bit FAT entries

Each cluster is referenced by a 12-bit value. 0x000 = free, 0xFF8–0xFFF = end of chain. Reading requires bit manipulation to extract odd/even 12-bit values from packed bytes.

Operations

// read file
fat12_read(name, buf, size);

// write / create file
fat12_write(name, buf, size);

// delete file
fat12_delete(name);  // entry[0] = 0xE5

// list root directory
fat12_ls();

// file name format: 8+3
// "HELLO TXT" (padded)
⏰ rtc.c — Real Time Clock
×

Reads time and date from the CMOS RTC via ports 0x70 (address) and 0x71 (data). CMOS stores values in BCD format — each byte encodes two decimal digits.

// read CMOS register
outb(0x70, reg);
val = inb(0x71);

// BCD → binary
bin = (bcd & 0x0F) + (bcd >> 4) * 10;

// timezone offset UTC+5
t->hour = (hour + 5) % 24;

Checks "Update in Progress" flag (bit 7 of register 0x0A) before reading to avoid getting corrupted mid-update values.

⏱ pit.c — Timer
×

The PIT (Programmable Interval Timer) chip fires IRQ0 at a configurable frequency. TEZ_OS programs it to fire at 100 Hz.

// PIT base frequency: 1193180 Hz
// divisor for 100 Hz:
#define PIT_DIVISOR (1193180 / 100)

outb(0x43, 0x36);  // mode
outb(0x40, lo);    // divisor low
outb(0x40, hi);    // divisor high

Every 100 ticks (1 second), the handler calls clock_update() which writes the current time directly to VGA memory at columns 61–79 of row 0.

🧮 TEZ Language — Compiler Internals
_
□
×

tez_lexer.c

Tokenizes source code into a flat array of Token structs. Handles keywords, identifiers, integer literals, string literals, char literals, and operators.

Identifiers are interned in a string table — the same name always gets the same index, which is critical for variable lookup in codegen.

Special: recognizes cons.out as a single token (dot notation).

tez_parser.c

Recursive descent parser that builds an AST (Abstract Syntax Tree) in a flat node pool. Each node has type, value, and up to 6 child indices.

Expression parsing respects full operator precedence: unary → mul → add → compare → equality → and → or.

Supports: var declarations, assignments, if/else, while, for, function declarations and calls, rtn, cons.out.

tez_codegen.c

Walks the AST and emits raw x86 32-bit machine code into an 8KB buffer. Uses standard cdecl calling convention with ebp frames.

Variables are stored on the stack at [ebp - slot*4]. Frame size is patched after the function body is emitted.

cons.out() calls kernel's vga_print/vga_putchar via absolute function pointer — MOV EAX, addr; CALL EAX.

⌨ keyboard.c — PS/2 Driver
_
□
×

PS/2 keyboard generates scan codes on port 0x60. Each keypress generates a "make" code; release generates "break" code (make | 0x80).

TEZ_OS uses two lookup tables — sc_normal[] and sc_shift[] — to translate scan codes to ASCII characters.

Shift and Caps Lock state is tracked. Characters are pushed to a 256-byte ring buffer — decoupling the IRQ handler from the shell read loop.

// ring buffer
static char buf[256];
static int head=0, tail=0;

// IRQ1 handler
uint8_t sc = inb(0x60);
if (sc & 0x80) {
  // key release
  if (sc==0xAA||sc==0xB6)
    shift = 0;
} else {
  // key press → ASCII
  char c = sc_normal[sc];
  buf_push(c);
}
